# ⚾ DUGOUT / BM — Baseball Manager Simulator

웹 기반 **8팀 KBO풍 야구 구단 경영 시뮬레이션**. 감독이 되어 타선·로테이션·불펜부터 드래프트·트레이드·FA·시설 투자·선수 육성까지 맡아 시즌을 반복 경영하며 우승을 노린다.

**[▶ 라이브 데모](https://chyoung001.github.io/Baseball_manager/)** · Vanilla JS · 빌드 도구 없음 · 브라우저에서 바로 실행

> 현행 구현(DUGOUT)을 살리면서 **Notion 설계서 BM v2**를 향해 점진 발전 중. 설계서가 게임 규칙의 단일 진실 공급원(SSOT)이며, 코드는 로드맵(아래)을 따라 단계적으로 그에 수렴한다.

브라우저 없이 Node `vm` + DOM/localStorage 스텁으로 전 모듈을 로드해 회귀를 검증한다.


## 5. 아키텍처 원칙

빌드 툴이 없으므로 **모든 함수·상수·상태가 전역 스코프에 공유**되고, 모듈 의존성은 오직 `index.html`의 `<script>` **선언 순서**로 보장된다.

- **단방향 데이터 흐름**: 단일 전역 상태 `G` → 렌더 함수 → `innerHTML` 주입. 상태 변경 후 관용적으로 `render*() → updateHeader() → saveGame()`으로 마무리.
- **화면 전환**: `.screen.active` CSS 토글 (라우터 없음).
- **네임스페이스**: 내부/사설 식별자는 언더스코어(`_potential`, `_compressPlayer`), CSS는 도메인 접두사(`bc-`, `dd-`, `lu-`).
- **저장**: `localStorage` 압축 스냅샷 + 버전 태그 `_v`. 로드 시 순차 마이그레이션(현재 `_v:5` — v4: 일반 스탯 20-80→1~100, v5: 히든 스탯 7~20→1~100 ×5 변환 + 신규 히든 6종 백필).

## 6. 핵심 시스템

### 6.1 시즌 흐름 (8페이즈 · 63경기)

```
🌸 프리시즌 → ⚾ 전반기(G1~30) → ⭐ 올스타 & 드래프트 → ⚾ 후반기(G31~63, G43~ 확대 엔트리)
→ 🏆 포스트시즌(4팀) → 🏅 시상식 & 은퇴 → 🗳️ GM 회의(룰 투표) → 🔥 스토브리그(FA)
```

- **정규시즌**: 63경기 = **21시리즈 × 3연전 고정**. 홈/원정은 시리즈 단위로 교대.
- **포스트시즌**: 상위 4팀 **균형 토너먼트** — 1v4·2v3 준PO(5전 3선승) → 챔피언십(7전 4선승).
- **GM 회의**: 시즌 말 8개 구단 룰 투표(과반 가결) → `SeasonModifiers`(사치세 라인·확대 엔트리 등)를 다음 시즌에 적용.

### 6.2 8개 팀 아키타입

| 팀 | 컨셉 | 정체성 | 예산 |
|---|---|---|---|
| 🪓 바이킹스 | power_hit | 파이어볼러 선발 왕국 | 100 |
| ⚔️ 세이버스 | pitching | 거대 자본의 악의 제국 | 160 |
| 🌟 드림즈 | prospect | 유망주 리빌딩(육성 90) | 75 |
| 🦅 이글스 | bullpen | 6회부터 잠그는 철벽 불펜 | 95 |
| 👯 트윈스 | speed | 도루·번트 스몰볼 | 85 |
| 😇 앤젤스 | sabermetrics | 출루율·FIP 세이버메트릭스 | 70 |
| 😈 데빌즈 | contact_hit | 머신건 컨택 타선 | 110 |
| 🐯 타이거즈 | defense | 센터라인 질식 수비 | 90 |

### 6.3 선수 데이터 모델 (1~100 스케일)

- **공개 능력치 (1~100, 50=평균 · 70+=엘리트)**
  - 타자: `contact, power, eye, speed, fielding, arm`
  - 투수: `stuff, control, velocity, movement, stamina, clutch`
- **히든 스탯 (1~100, 50=평균 · P2-2에서 10종 확장)**: 잠재력 `_potential`(별도 축) + 공통 8종 `_durability`(부상위험) `_consistency`(일관성) `_clutchHidden`(클러치) `_workEthic`(프로의식) `_versatility`(다재다능) `_ambition`(야망) `_loyalty`(충성심) `_temperament`(참을성) + 투수 `_recovery`(연투회복) / 타자 `_pullTendency`(당김성향)
  - 협상 연동: 야망 → 요구액 ±10%, 충성심 → 재계약(재적 3년+) 홈타운 디스카운트 최대 12.5% (야망이 크면 상쇄), 참을성 → 협상 기회 2~4회
  - 성장 연동: 프로의식·야망 모두 75+ 시 "엘리트 성장 곡선" (XP 성장 +20%), 연투회복 → 투수 미등판 회복 ±2
  - 다재다능 → 포지션 전환 페널티 가감 (아래 서브 포지션) · 당김성향은 값만 생성 (P4 매치엔진에서 활용 예정)
- **서브 포지션 (P2-1)**: 타자 생성 시 0개 60% / 1개 35% / 2개 5% (인접 포지션만, C·DH 제외). 다재다능 65+ 타자는 프리시즌마다 10% 확률로 커리어 중 최대 1개 추가 습득.
  - **다재다능 세금**: 서브 1개 OVR −1 / 2개 −2
  - **전환 페널티 (비대칭, 수비 스탯 %)**: 쉬움 −5% (2B↔SS, LF↔RF, LF/RF/3B→1B) · 보통 −12% (SS→3B, CF→코너, 2B→3B) · 어려움 −22% (OF↔IF, 역방향) · **→C 불가** · DH는 슬롯(무페널티). 서브 포지션 경험 시 절반, 다재다능 히든으로 ±50% 가감. 매치 엔진 수비력 계산에 `effFielding/effArm` 반영.
  - **가중치 힌트 (분석팀 레벨)**: L1 핵심 스탯 TOP 2 → L2 전 스탯 4단계 가중치 → L3 포지션 전환 OVR 시뮬
- **OVR (P2-1, 이원화)**:
  - `ovrRaw(p)` — **절대 점수**. 포지션 그룹별(타자 C/MIF/CIF/OF/DH, 투수 SP/CP/SU/MR/LR) 가중 평균. 생성·성장캡·에이징 등 "물리적 스탯" 맥락 전용.
  - `ovr(p)` — **상대 OVR**. 1군 리그 분포 대비 Z-score(50=리그평균, σ=15) + 다재다능 세금(서브1 −1 / 서브2+ −2). 표시·연봉·FA·트레이드 등 "가치" 맥락 전용. 등급 밴드 **S[84~100] · A[67~82] · B[51~65] · C[34~49] · D[9~32]**
- **잠재력 상한**: `maxOVR = floor(18 + potential × 0.825)` (pot 50→59, pot 100→100) — **raw 기준** 판정
- **생성 등급 분포**: S 3% · A 15% · B 42% · C 30% · D 10% (1군 주전 OVR 중앙값 ≈ 50, 2군/육성은 낮게 유지)

#### 표시 스케일 (프론트오피스 레벨별 포그오브워)
분석팀 레벨에 따라 **타 구단·FA 선수**의 능력치 공개 정밀도가 4단계로 차등된다 (본인 팀은 항상 정확).

| 레벨 | 분석팀 Lv | 표시 |
|---|---|---|
| **L0** | <20 | 등급 문자 (S/A/B/C/D) |
| **L1** | 20~39 | 5단위 버킷 (예: 80~84) |
| **L2** | 40~59 | ±추정 (예: 81~87) |
| **L3** | ≥60 | 정확값 |

### 6.4 경기 시뮬레이션 엔진

매 타석은 **TTO(Three True Outcomes) → 인플레이** 2단계 확률 모델. 능력치 1~100 확대에 맞춰 스프레드 계수를 재보정(÷1.65)해 리그 평균은 MLB 근사(타율 ~0.27)로 앵커하고 개인 극단값은 현실 범위로 억제한다.

**1차 TTO 확률** (기본값 앵커 + 능력차 스프레드, 클램프):

| 결과 | 공식 | 기본 | 범위 |
|---|---|---|---|
| 홈런 | `0.025 + (파워 − 무브먼트) / 165 × 0.16` | 2.5% | 0.5~8% |
| 삼진 | `0.180 + (구위 − 컨택) / 165 × 0.15` | 18% | 4~30% |
| 볼넷 | `0.090 + (선구안 − 제구) / 165 × 0.12` | 9% | 2~15% |

**2차 인플레이**: 수비 에러(0.5~4%) → BABIP 안타(컨택×수비×회귀) → 범타. 안타 시 타구 유형은 3루타(speed) → 2루타(power) → 단타 순.

**주루** (주자 객체의 실제 `speed`로 판정): 단타 2루→홈, 1루→3루(speed>75), 2루타 1루→홈(speed>59), 희생플라이 `0.50+(speed−50)/330`, 도루(speed>67, 팀 컨셉 배율), 병살(speed≤42 1.4배 / ≥75 0.6배).

**투수 피로도**: 50구부터 vel/ctrl/mov 선형 감산(최대 −12/−12/−10) + 잔여 스태미나 배율(×1.0~0.40). 최대 투구수 = 선발 `stamina+40`, 불펜 `stamina/2+10`. 교체는 보직 우선순위(CP/SU/MR/LR) 기반.

**성장(XP)**: 경기당 XP(선발 10 / 벤치 3 / 2군 7 / 육성 5) → 레벨업 요구치 OVR 밴드별(≥84:250 / ≥67:150 / ≥51:110 / ≥34:90 / 그 외:70). 성장 시 랜덤 능력치 +`rand(1,3)×workEthic 배율`, `maxOVR` 도달 시 차단.

**동적 회귀 보정**: 시즌 중 타율 >0.380(30타수+) 시 안타 확률 −20%, >0.350 시 −10%로 비현실 성적 억제.

### 6.5 부상 · 컨디션 · 슬럼프

- **부상**: 경기 중/후 컨디션·내구성·투구수 기반 확률 롤 → IL. 유형 가중 랜덤 (경미 50% 3~7경기 / 중등도 35% 8~18 / 중증 12% 20~40 / 시즌아웃 3% = 63경기). 회복: IL → 2군 재활(−15 디버프) → 쿨다운 → 콜업, 복귀 후 재부상 위험 1.5배.
- **컨디션**: 출전 소모 / 미출전·2군 회복 / 연투 패널티. 슬럼프는 `max(1, 15−consistency)`% 확률로 발동, 컨디션 40 미만 시 contact/power −12.
